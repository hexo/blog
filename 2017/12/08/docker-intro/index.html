docker 相关知识总结

发表于 2017-12-08 | 分类于 linux |

最近项目用到了docker相关的一些技术，感觉还是挺方便的，有一些学习心得，记录一下

docker 常用命令

  • docker ps ：可以查看正在运行着的docker容器
  • docker ps -a : 查看系统中已经存在的docker容器
  • docker images : 查看docker镜像
  • service docker restart : 重启docker服务

加载镜像

那么，第一步，我们需要加载一个docker镜像到系统中，可以通过导入镜像文件的方式，也可以在dockerhub上pull下来别人上传的镜像

从DockerHub pull 镜像

具体的选项可以通过 docker pull –help 命令看到，这里我们说一下镜像名称的格式。

  • Docker 镜像仓库地址：地址的格式一般是 <域名/IP>[:端口号]。默认地址是 Docker Hub。
  • 仓库名：如之前所说，这里的仓库名是两段式名称，即 <用户名>/<软件名>。对于 Docker Hub，如果不给出用户名，则默认为 library，也就是官方镜像。

生成容器

到了这一步，镜像的加载已经成功了，我们可以在Terminal输入docker images 查看当前已经存在的镜像，下一步就是生成容器了

docker run 命令可以加入参数，主要用到的如下：

  • -it：这是两个参数，一个是 -i：交互式操作，一个是 -t 终端。我们这里打算进入 bash 执行一些命令并查看返回结果，因此我们需要交互式终端。
  • –name container_name : 声明容器名字，如果不加入此项内容，每次启动容器时需要使用container_id
  • -d ：后台启动，使用这条命令会将容器运行在后台环境
  • -p docker_port:outer_port ：端口映射，docker内部端口映射到外部，使得服务可以被访问

删除镜像

主要有以下几个命令，方便镜像和容器的删除操作

  • docker kill $(docker ps -a -q) : 杀死正在运行的容器
  • docker rm $(docker ps -a -q) : 删除所有已经停止的容器
  • docker rmi $(docker images -q) : 删除所有镜像
